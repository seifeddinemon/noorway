# دليل بناء نظام iOS من Windows (NoorFaith)

نظرًا لأن بناء تطبيقات iOS يتطلب نظام macOS وبرنامج Xcode، فقد قمنا بإعداد **GitHub Actions** ليقوم بعملية البناء تلقائيًا في السحابة.

## خطوات الحصول على نسخة iOS (IPA)

1. **ارفع الكود إلى GitHub:**
   تأكد من دفع (Push) التغييرات الخاصة بك إلى مستودع GitHub الخاص بالمشروع.
   
2. **شغل الـ Workflow:**
   - انتقل إلى تبويب **Actions** في مستودع GitHub الخاص بك.
   - اختر **iOS Build** من القائمة اليسرى.
   - إذا كنت تريد تشغيله يدويًا، اضغط على **Run workflow**.

3. **تحميل الملف:**
   - بمجرد اكتمال البناء (يتحول اللون إلى الأخضر)، اضغط على اسم المهمة.
   - في الأسفل ستجد قسم **Artifacts**، قم بتحميل ملف `ios-ipa`.

## ملاحظات هامة حول التوقيع (Signing)

- الملف الناتج حاليًا هو نسخة **Unsigned** (بدون توقيع)، وهي مناسبة للفحص أو إذا كان لديك جهاز Mac لاحقًا للتوقيع.
- لرفع التطبيق إلى App Store، ستحتاج إلى:
  - حساب Apple Developer.
  - إضافة شهادات التوقيع (Certificates) و (Provisioning Profiles) كـ **GitHub Secrets**.
  - تحديث ملف `ios-build.yml` لاستخدام هذه الأسرار.

> [!TIP]
> إذا واجهت أي مشاكل في البناء، تحقق من سجلات (Logs) المهمة في GitHub Actions لمعرفة الخطأ بالتحديد.
